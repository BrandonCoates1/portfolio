Morse Code Translator

This allows you to input morse code or English,
which then translates either to English or morse code.

I built this with python and learned how to use a
dictionary. This allowed me to store the English alphabet
into a key: value pair with the respective morse code.

As you can see in the message() function, this checks what
the user inputs and returns either true or false.

The translate() function then creates a reversed dictionary
(for use in translating morse code to English), takes the
boolean of message() and translates to morse code if true,
and to English if false: returning the list (trans) as a string.

Text Based Adventure Game

This was created with python with another person. We had 1 week
to complete the project. I worked on stuff like what you can see
to the left.

The function create_class() asks the user multiple questions to
decideon what class the player gets. This then gets inputted into
a class (class is like a blueprint).

The story of the game is that you wake up in a rundown barn, and
you decide where you want to go. You have 3 options, each option
takes you through different paths, and leads to multiple endings.

The game has turn based combat against a variety of different
enemies, which has its' own randomised stats.

Bank Note Withdraw

This was one of the first challenges I did when learning python.
The challenge was to take a number between 40 and 4001,
and return the smallest amount of £100, £50 and £20 notes
the atm would dispense.

first, it checks if the input is a valid number between 40 and 4001,
then splits the number into each respective column [1000, 100, 10].

This is then run through the withdraw() function to do maths
depending on the size of the list created earlier, and returns
the smallest amount of each note.

Overall, it works but the code can definitely be refined to be
more effective and efficient. That is one thing I would do
differently, if I did it again.
